# Synthesis 제품 정의

Synthesis는 준비 없이 시작한 대면 회의가 표면 문제와 뻔한 솔루션에서 멈추지 않도록, 팀의 날것 생각과 공공데이터를 `표면 → 본질`의 문제정의로 가공하는 현장 세션 도구다. 목표는 회의가 끝나는 순간 팀원과 외부인이 즉시 이해할 수 있는, 데이터 근거가 연결된 본질 문제정의 문서를 남기는 것이다.

## 설계 원리

> 직감·판단은 인간, 그 외 모든 노동은 AI.

- AI는 팀 입력 구조화, MECE 원인 가설 발산, 공공데이터 검색, 최종 문서 편집을 맡는다.
- 사람은 본질에 가깝다고 느끼는 문제 가지와 실제 정의에 쓸 데이터 근거를 명시적으로 선택한다.
- 사람의 선택이 없는 문제 가지나 근거를 AI가 최종 정의에 몰래 포함하지 않는다.

## 현장 흐름

1. **표면 포착** — `[누가][상황][문제][영향]`을 채우고 각자 날것 생각을 먼저 공용 보드에 적는다.
2. **MECE 발산** — AI가 표면 문제를 하나의 축으로 MECE 분해하고 다음 `왜?` 질문을 만든다.
3. **데이터 검증** — 각 문제 가지에 공식·공공 데이터 근거를 검색해 붙인다. 팀원이 찾은 자료도 같은 형식으로 직접 추가한다.
4. **직감 수렴** — 팀원은 본질 후보와 채택 근거를 각자 선택한다. 데이터는 가설 지지뿐 아니라 새 가지 촉발과 반증에도 사용한다.
5. **본질 정의** — 선택된 모든 핵심 가지에 근거가 있고, 새 발견 또는 반증 근거가 하나 이상 있을 때 하나의 문제정의 문서로 저장한다.

## 성공 조건

- 최종 문서에 본질 원인과 표면에서 본질까지의 Why chain이 드러난다.
- 사람이 선택한 모든 핵심 문제 가지마다 채택 데이터 근거가 하나 이상 연결된다.
- 데이터로 새로 발견하거나 기각한 문제가 하나 이상 명시된다.
- 외부인이 사전 설명 없이 읽을 수 있는 `[누가][상황][문제][영향]` 문제 문장을 제공한다.
- 팀원 입력, 본질 투표, 근거 채택, 최종 문서가 프로젝트별로 실시간 축적·저장된다.

## 이번 범위에서 하지 않는 것

- 시장조사와 솔루션 설계
- 근거 없는 백지 문제 생성
- AI가 사람의 직감과 본질 판단을 대신하는 흐름
- 비대면 원격 회의에 최적화된 진행 기능
